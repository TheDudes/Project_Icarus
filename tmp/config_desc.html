 44 status_time: time,type [long][string]
 45 
 46     this key is to specify a duration after which a little status message
 47     should be displayed. this statusmessaged displays how many times the
 48     interpreter went in takt, how many times not, and avg time for the last
 49     evaluations. if no type is specified, time is assumed in seconds.
 50 
 51     time: number between 0 and max long, if 0 is specified, the status will
 52           not be printed.
 53 
 54     type: s/seconds -> seconds
 55           m/minutes -> minutes
 56           h/hours   -> hours
 57 
 58     default:
 59 
 60     examples: - 20s -> display status every 20 seconds
 61               - 20m -> display status every 20 minutes
 62               -  2h -> display status every  2 hours
 63               -  0  -> do not display any status

196 verbosity_level: verbosity [int]
197 
198     a value to specify how verbose the log messages are, a higher level stands
199     for an more verbose logging (more logs), a lower level for less logging.
200     this option is for debugging.
201 
202     verbosity: - 0 -> just log important output/ERRORs (these messages are also
203                          printed to console if the silent flag is set to false)
204                - 1 -> be a bit more verbose, show some information (PRINT
205                          messages are also verbosity 1)
206                - 2 -> even more verbose, run times are verbosity 2 for example.
207                - 3 -> here starts a more debugging like output, not recommended
208                          for normal usage.
209                - 4 -> debug mode, log files are getting rly huge here.
210 
211     default:
212 
213     examples: -
